# AI生成测试代码质量保障方案

## 问题分析

AI生成的测试代码存在不确定性，可能无法直接执行的原因包括：

1. **语法错误** - 生成的代码可能有语法问题
2. **导入缺失** - 缺少必要的import语句
3. **断言不合理** - 生成的断言逻辑可能不正确或不完整
4. **变量/方法不存在** - 引用了未定义的变量或方法
5. **不符合项目规范** - 代码风格与项目不一致
6. **依赖缺失** - 使用了项目未安装的第三方库

## 解决方案

### 方案一：代码验证与自动修正层（推荐）

在生成代码后、保存前增加验证和修正环节：

#### 1. 语法验证模块
- 使用Python AST模块进行语法检查
- 检测语法错误并定位问题位置
- 返回详细的错误信息

#### 2. 静态代码分析
- 集成pylint或flake8进行代码质量检查
- 检测未定义变量、未使用导入等问题
- 生成问题报告

#### 3. 自动修正机制
- 自动补充缺失的标准导入（requests, pytest, allure等）
- 修复常见的简单语法错误
- 使用AI进行自我修正（将错误信息反馈给AI重新生成）

#### 4. 沙箱预执行
- 在隔离环境中执行生成的代码（仅导入检查，不实际运行测试）
- 捕获ImportError和NameError
- 提供详细的错误反馈

### 方案二：模板约束生成

通过更严格的模板约束AI生成内容：

#### 1. 增强Prompt模板
- 在Prompt中明确要求生成可执行代码
- 提供完整的代码示例作为参考
- 添加代码检查清单

#### 2. 代码片段模板
- 预定义常用的代码片段（请求模板、断言模板等）
- AI只需填充关键参数，而非生成完整代码
- 减少生成不确定性

#### 3. 结构化输出
- 要求AI按固定JSON结构输出代码各部分
- 分离导入、函数定义、断言等模块
- 便于验证和组装

### 方案三：迭代优化机制

建立生成-验证-修正的闭环：

#### 1. 错误反馈循环
- 验证失败时，将错误信息反馈给AI
- AI根据错误信息重新生成或修正代码
- 设置最大重试次数（如3次）

#### 2. 人工审核机制
- 对于无法自动修正的问题，标记待审核状态
- 提供可视化差异对比工具
- 支持人工编辑和确认

#### 3. 质量评分
- 对生成的代码进行质量评分
- 记录历史生成质量，优化Prompt

## 实施计划

### 第一阶段：代码验证层（核心）

**新增文件：**
- `src/core/code_validator.py` - 代码验证器核心模块

**功能实现：**
1. `SyntaxValidator` - 语法验证器
   - 使用AST检查语法
   - 返回语法错误位置和描述

2. `ImportValidator` - 导入验证器
   - 检查必需导入是否存在
   - 自动补充缺失的标准导入

3. `StaticAnalyzer` - 静态分析器
   - 检测未定义变量
   - 检测未使用导入
   - 检测潜在问题

4. `CodeFixer` - 代码修正器
   - 自动修复简单问题
   - 生成修正建议

**集成点：**
- `test_script_service.py` 的 `_generate_with_ai` 方法中
- 生成代码后立即验证
- 验证失败时触发修正流程

### 第二阶段：AI自我修正

**修改文件：**
- `src/ai/prompts.py` - 添加修正Prompt模板
- `src/core/test_script_service.py` - 添加重试逻辑

**功能实现：**
1. `CodeFixPrompt` - 代码修正Prompt
   - 包含原始代码和错误信息
   - 要求AI输出修正后的完整代码

2. 重试机制
   - 验证失败时调用AI修正
   - 最多重试3次
   - 记录修正历史

### 第三阶段：沙箱预执行

**新增文件：**
- `src/core/sandbox_executor.py` - 沙箱执行器

**功能实现：**
1. 隔离执行环境
   - 使用独立的命名空间
   - 仅执行导入和语法检查
   - 不实际发送HTTP请求

2. 错误捕获
   - 捕获ImportError
   - 捕获NameError
   - 捕获SyntaxError

### 第四阶段：UI增强

**修改文件：**
- `src/ui/pages/test_script_page.py` - 测试脚本页面

**功能实现：**
1. 验证状态显示
   - 显示验证通过/失败状态
   - 展示错误详情
   - 提供修正建议

2. 代码预览增强
   - 语法高亮显示错误行
   - 差异对比视图
   - 一键应用修正

## 预期效果

1. **提高代码可执行率** - 从不确定状态提升到90%+可直接执行
2. **减少人工干预** - 自动修正大部分常见问题
3. **提供透明反馈** - 用户清楚知道代码质量状态
4. **持续优化** - 通过历史数据优化生成质量

## 依赖要求

无需新增外部依赖，使用Python标准库：
- `ast` - 语法分析
- `compile` - 编译检查
- `exec` - 沙箱执行

可选增强（如需更强大的静态分析）：
- `pylint` - 代码质量检查
- `flake8` - 风格检查
