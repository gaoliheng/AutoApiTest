## 1. 项目概述

### 1.1 项目简介

AutoApiTest 是一个基于 AI 的接口自动化测试平台，旨在帮助测试人员快速生成和管理 API 测试用例，并自动生成符合 pytest + Allure 框架规范的测试脚本。通过集成 AI 模型，系统能够根据接口文档智能生成测试用例，大幅提升测试效率。

### 1.2 项目目标

- 提供直观的图形化界面，简化测试用例管理流程
- 利用 AI 技术自动生成测试用例，减少人工编写工作量
- 支持多种 AI 模型配置，满足不同用户需求
- 自动生成符合规范的测试脚本，提高测试代码质量
- 提供代码验证和修复功能，确保生成的代码可用性
- 支持测试数据和脚本的导出，便于团队协作

### 1.3 目标用户

- API 测试工程师
- 自动化测试开发人员
- 质量保证(QA)团队
- 需要进行接口测试的开发人员

### 1.4 应用场景

- RESTful API 接口测试
- 微服务接口测试
- 接口回归测试
- 接口性能测试准备
- 持续集成(CI/CD)测试集成

## 2. 功能需求

### 2.1 AI 模型配置管理

#### 2.1.1 功能描述

提供 AI 模型的配置管理功能，支持添加、编辑、删除、测试连接等操作，用户可以配置多个 AI 模型并设置默认模型。

#### 2.1.2 功能需求

| 需求编号 | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-AI-001 | 支持添加新的 AI 模型配置 | 高 |
| FR-AI-002 | 支持编辑已存在的 AI 模型配置 | 高 |
| FR-AI-003 | 支持删除 AI 模型配置 | 中 |
| FR-AI-004 | 支持测试 AI 模型连接，显示连接状态和延迟 | 高 |
| FR-AI-005 | 支持设置默认 AI 模型，同一时间只能有一个默认模型 | 高 |
| FR-AI-006 | 支持查看所有已配置的 AI 模型列表 | 高 |
| FR-AI-007 | AI 模型配置信息包括：名称、API 地址、API Key、模型标识 | 高 |

#### 2.1.3 数据字段

- **模型名称**：模型的显示名称（如：GPT-4、Claude-3）
- **API 地址**：AI 服务的 API 基础 URL（如：https://api.openai.com/v1）
- **API Key**：访问 AI 服务的密钥
- **模型标识**：具体的模型名称（如：gpt-4、claude-3-opus）
- **是否默认**：标记是否为默认使用的模型
- **创建时间**：配置创建的时间戳
- **更新时间**：配置最后更新的时间戳

### 2.2 登录配置管理

#### 2.2.1 功能描述

提供接口登录认证的配置管理功能，支持配置多个登录接口，用于获取认证 Token，支持 Token 提取和请求头配置。

#### 2.2.2 功能需求

| 需求编号 | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-AUTH-001 | 支持添加新的登录配置 | 高 |
| FR-AUTH-002 | 支持编辑已存在的登录配置 | 高 |
| FR-AUTH-003 | 支持删除登录配置 | 中 |
| FR-AUTH-004 | 支持测试登录配置，验证配置是否正确 | 高 |
| FR-AUTH-005 | 支持启用/禁用登录配置 | 中 |
| FR-AUTH-006 | 支持 JSONPath 表达式提取 Token | 高 |
| FR-AUTH-007 | 支持配置 Token 前缀（如：Bearer） | 中 |
| FR-AUTH-008 | 支持自定义 Header 名称 | 中 |
| FR-AUTH-009 | 支持查看所有登录配置列表 | 高 |
| FR-AUTH-010 | 在测试用例页面支持一键获取 Token | 高 |

#### 2.2.3 数据字段

- **配置名称**：登录配置的显示名称
- **Base URL**：接口的基础地址
- **登录路径**：登录接口的路径（如：/api/auth/login）
- **请求方法**：HTTP 请求方法（GET/POST/PUT/PATCH）
- **请求头**：登录请求的额外请求头（JSON 格式）
- **请求体**：登录请求的请求体（JSON 格式）
- **Token 路径**：从响应中提取 Token 的 JSONPath 表达式
- **Token 前缀**：Token 的前缀（默认：Bearer）
- **Header 名称**：认证 Header 的名称（默认：Authorization）
- **是否启用**：标记配置是否启用
- **创建时间**：配置创建的时间戳
- **更新时间**：配置最后更新的时间戳

| FR-TC-004 | 支持清空所有测试用例 | 中 |

| FR-TC-006 | 支持通过 AI 根据接口文档生成测试用例 | 高 |

#### 2.3.3 数据字段

- **用例名称**：测试用例的名称
- **接口路径**：API 接口的路径
- **请求方法**：HTTP 请求方法（GET/POST/PUT/DELETE/PATCH）
- **请求头**：请求头信息（字典格式）
- **请求参数**：查询参数或路径参数（字典格式）
- **请求体**：请求体数据（JSON 格式）
- **预期状态码**：预期的 HTTP 状态码（默认：200）
- **断言**：测试断言列表
- **创建时间**：测试用例创建的时间戳
- **更新时间**：测试用例最后更新的时间戳

#### 2.3.4 AI 生成测试用例

系统应支持通过 AI 根据接口文档自动生成测试用例，包括：

- 从接口文档中提取 Base URL 和接口路径
- 从接口文档中提取公共请求头（特别是认证信息）
- 为接口生成多个测试场景：
  - 正常场景测试
  - 异常场景测试
  - 边界值测试
- 支持自然语言描述的请求参数、请求体和断言
- 生成的测试用例应包含完整的测试信息

### 2.4 测试脚本生成

#### 2.4.1 功能描述

根据测试用例自动生成符合 pytest + Allure 框架规范的 Python 测试脚本，支持代码验证和 AI 自动修复功能。

#### 2.4.2 功能需求

| 需求编号 | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-TS-001 | 支持根据测试用例生成 Python 测试脚本 | 高 |
| FR-TS-002 | 生成的脚本使用 pytest 框架 | 高 |
| FR-TS-003 | 生成的脚本包含 Allure 注解 | 高 |
| FR-TS-004 | 支持使用 httpx 库发送 HTTP 请求 | 高 |
| FR-TS-005 | 每个测试用例生成独立的测试函数 | 高 |
| FR-TS-006 | 支持代码语法验证 | 高 |
| FR-TS-007 | 支持自动修复常见的代码错误 | 高 |
| FR-TS-008 | 支持 AI 智能修复代码问题 | 中 |
| FR-TS-009 | 支持保存生成的脚本为文件 | 高 |
| FR-TS-010 | 支持复制脚本到剪贴板 | 中 |
| FR-TS-011 | 支持在界面中查看和编辑生成的脚本 | 高 |
| FR-TS-012 | 提供测试脚本执行帮助文档 | 中 |

#### 2.4.3 代码规范要求

生成的测试脚本应遵循以下规范：

- 使用 pytest 框架编写测试函数
- 使用 allure-pytest 添加 Allure 注解：
  - @allure.feature：功能模块名称
  - @allure.story：用户故事或场景
  - @allure.severity：严重程度（blocker/critical/normal/minor/trivial）
  - @allure.title：测试用例标题
  - @allure.description：测试用例描述
  - @allure.step：测试步骤
- 使用 httpx 库发送 HTTP 请求
- 包含完整的断言验证
- 代码需要有适当的注释
- 遵循 PEP 8 编码规范

#### 2.4.4 代码验证功能

系统应提供以下代码验证功能：

- 语法验证：检查 Python 语法错误
- 导入检查：检查缺失的 import 语句
- 静态分析：检查未定义的变量
- 沙箱执行：在安全环境中执行代码检查运行时错误
- 自动修复：自动添加缺失的导入语句
- AI 修复：调用 AI 模型修复代码问题

## 4. 技术架构

| token_path | TEXT | NOT NULL DEFAULT '$.data.access_token' | Token 路径 |
| token_prefix | TEXT | DEFAULT 'Bearer' | Token 前缀 |
| header_name | TEXT | DEFAULT 'Authorization' | Header 名称 |
| is_enabled | INTEGER | DEFAULT 1 | 是否启用 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

## 6. 接口规范

### 6.1 AI 模型接口

#### 6.1.1 连接测试

**接口描述**：测试 AI 模型连接是否正常

**请求参数**：
- model_name: 模型名称
- api_base: API 基础 URL
- api_key: API Key

**响应结果**：
- success: 是否成功
- message: 结果消息
- model_info: 模型信息
- latency_ms: 延迟（毫秒）

#### 6.1.2 聊天接口

**接口描述**：调用 AI 模型进行对话

**请求参数**：
- model: 模型名称
- messages: 消息列表
- max_tokens: 最大 token 数
- temperature: 温度参数

**响应结果**：
- choices: 选择列表
- usage: 使用情况
- model: 使用的模型

### 6.2 登录认证接口

#### 6.2.1 执行登录

**接口描述**：执行登录请求获取 Token

**请求参数**：
- base_url: 基础 URL
- login_path: 登录路径
- method: 请求方法
- headers: 请求头
- body: 请求体

**响应结果**：
- success: 是否成功
- token: Token 值
- header_name: Header 名称
- header_value: Header 值
- raw_response: 原始响应
- error_message: 错误消息
- status_code: 状态码

## 8. 部署要求

### 8.1 系统环境要求

| 项目 | 要求 |
|-----|------|
| 操作系统 | Windows 10/11 |
| Python 版本 | Python 3.8 或更高版本 |
| 内存 | 最低 4GB，推荐 8GB |
| 硬盘 | 最低 500MB 可用空间 |
| 网络 | 需要连接互联网（调用 AI API） |

### 8.2 依赖包

```
PyQt6>=6.5.0
httpx>=0.25.0
openai>=1.0.0
pyyaml>=6.0
pytest>=7.4.0
allure-pytest>=2.13.0
pyinstaller>=6.0.0
jsonpath-ng>=1.6.0
```

### 8.3 安装步骤

1. 安装 Python 3.8 或更高版本
2. 安装依赖包：`pip install -r requirements.txt`
3. 运行程序：`python src/main.py`

### 8.4 打包发布

使用 PyInstaller 打包为独立可执行文件：

```bash
cd build_scripts
python build.py
```

打包完成后，可执行文件位于 `dist/AutoApiTest.exe`。
